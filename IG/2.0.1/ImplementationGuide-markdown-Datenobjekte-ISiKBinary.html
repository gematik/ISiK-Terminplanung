ISiKBinary (Binary)

FHIR-Profil

Es gelten bis auf weiteres die Vorgaben des ISiK Basismoduls für das Profi 'ISiKBinary'. Ein Termin Repository MUSS die Anlage (Create-Interaktion) einer Binary-Ressource entsprechend der Vorgaben des ISiK-Basismoduls unterstützen. Die Binary-Ressource MUSS in ihrem originalen Repräsentationsformat, sowie als FHIR-Ressource abgerufen werden können (vgl. Serving Binary Resources using the RESTful API).
